# 사용자 승인 정책

## 목적

도면 데이터와 개발 환경을 손상시킬 수 있는 작업은 AI가 임의로 수행하지 않고 먼저 사용자에게 승인받는다.

## 반드시 승인받을 작업

- 파일 또는 폴더 삭제
- 파일 이동, 이름 변경, 덮어쓰기
- 프로젝트 저장 형식 또는 DB schema 변경
- 기존 데이터의 일괄 변환
- 외부 crate, SDK, 시스템 프로그램 추가
- 네트워크 사용 또는 외부 서버 전송
- 인증서, 토큰, 비밀번호, 개인정보 처리
- GUI framework, rendering backend 교체
- crate 구조를 크게 변경하는 작업
- 기존 테스트를 삭제하거나 검증 기준을 낮추는 작업

## 승인 없이 가능한 작업

- 새 단위 테스트 추가
- 순수 계산 함수 추가
- 문서 작성 및 수정
- 컴파일, 포맷, 정적 분석, 테스트 실행
- 새 파일을 안전한 프로젝트 폴더에 추가
- 오류 메시지 개선

## 승인 요청 형식

```text
승인이 필요한 작업: [작업명]
변경 대상: [파일, 폴더, 의존성]
필요한 이유: [목적]
예상 영향: [데이터, 보안, 빌드, 호환성]
되돌리는 방법: [복구 방법]
```

## 보안 원칙

- 비밀값을 소스, 로그, 문서에 기록하지 않는다.
- 사용자 도면을 네트워크로 보내지 않는다.
- 파일 작업 전 대상 경로를 명확히 확인한다.
- 임시 파일과 백업 파일의 위치를 명확히 한다.
- 파괴적 작업은 dry-run 또는 확인 단계를 제공한다.
