# 아키텍처 정책

## 핵심 원칙

UI는 도면 데이터를 직접 변경하지 않는다. 모든 변경은 command를 거쳐 core model에 적용한다.

```text
사용자 입력 -> UI 이벤트 -> DrawingCommand -> Core Model -> 검증 -> 렌더 모델 -> 화면/출력/저장
```

## Workspace 구성

```text
cad_studio/
├─ Cargo.toml
├─ docs/
└─ crates/
   ├─ cad_core/
   ├─ cad_geometry/
   ├─ cad_tolerance/
   ├─ cad_command/
   ├─ cad_render/
   ├─ cad_io/
   ├─ cad_batch/
   ├─ cad_cli/
   └─ cad_app/
```

## crate 책임

- `cad_core`: Project, Drawing, Entity, Layer, Dimension, stable ID
- `cad_geometry`: 점, 선, 원, 호, 교차, 거리, 변환, 스냅
- `cad_tolerance`: 공차 범위, 누적, 끼워맞춤, 계산 trace
- `cad_command`: 실행, 검증, undo, redo, 재생
- `cad_render`: core model을 RenderPrimitive로 변환
- `cad_io`: 저장, 불러오기, 백업, 마이그레이션, 출력
- `cad_batch`: 반복 도면과 대량 출력
- `cad_cli`: GUI 없이 검증, 출력, 복구
- `cad_app`: UI와 입력 처리

## 의존성 방향

```text
cad_app -> cad_command -> cad_core
cad_command -> cad_geometry
cad_cli -> cad_io -> cad_core
cad_batch -> cad_command / cad_io / cad_core
cad_render -> cad_core
cad_geometry -> cad_core
cad_tolerance -> cad_core
```

core는 UI, DB, 렌더링 backend에 의존하지 않는다.

## 변경 승인 대상

다음은 사용자 승인 없이 변경하지 않는다.

- crate 구조
- 저장 형식과 schema
- command 체계
- stable ID 정책
- 단위 및 부동소수점 정책
- GUI framework 또는 rendering backend
