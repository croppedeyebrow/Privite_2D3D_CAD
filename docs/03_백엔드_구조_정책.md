# 백엔드 구조 정책

## 단위

내부 계산 단위는 항상 밀리미터다. 화면 표시 단위와 계산 단위를 분리하며, 단위 변환은 명시적인 함수에서만 수행한다.

```rust
struct LengthMm(f64);
```

## Core Model

모든 영속 객체는 안정적인 ID를 갖는다.

- ProjectId
- DrawingId
- EntityId
- LayerId
- DimensionId
- SheetId

초기 Entity는 Line, Polyline, Rectangle, Circle, Arc, Text를 대상으로 한다.

## Geometry

- 순수 함수로 구현한다.
- 부동소수점 직접 비교를 금지한다.
- epsilon을 명시한다.
- zero-length line, 음수 반지름, 잘못된 호 각도는 validation error다.
- 좌표와 출력 정렬 순서는 deterministic해야 한다.

## 공차

초기 지원 타입은 다음과 같다.

```rust
enum ToleranceSpec {
    None,
    Symmetric,
    Bilateral,
    Limit,
}
```

계산 결과는 nominal, min, max를 포함해야 하며 모든 계산은 trace를 남긴다.

공차 표시용 반올림값을 내부 계산에 재사용하지 않는다. MVP 누적 계산은 worst-case 방식으로 한다.

## 검증

오류와 경고를 구분한다.

- 오류: 저장, 출력, 명령 실행을 막는 문제
- 경고: 작업은 계속할 수 있지만 사용자가 확인해야 하는 문제

검증 결과는 대상 객체 ID, 문제 설명, 해결 제안을 포함한다.

## 저장과 출력

저장 순서는 다음과 같다.

```text
메모리 검증 -> 임시 파일 작성 -> flush -> 재검증 -> 원자적 교체 -> 백업 상태 갱신
```

저장 형식 변경과 외부 의존성 추가는 `05_사용자_승인_정책.md`에 따라 승인받는다.
