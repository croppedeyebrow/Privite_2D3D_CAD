# AI 개발 작업 명령서

이 문서는 Codex, Claude, Cursor 등 모든 AI 개발 도구에 공통으로 전달한다.

## 작업 시작 전

1. `docs/00_문서_목차.md`를 읽는다.
2. 현재 작업과 관련된 정책 문서를 읽는다.
3. 프로젝트 구조와 기존 변경 사항을 확인한다.
4. 변경 대상과 완료 조건을 먼저 요약한다.
5. 승인 대상 작업인지 `docs/05_사용자_승인_정책.md`로 판단한다.

## 구현 규칙

- 기존 crate 책임을 우선한다.
- UI에서 core model을 직접 수정하지 않는다.
- 도면 상태 변경은 command를 사용한다.
- 계산은 결정적이어야 한다.
- 내부 길이 단위는 밀리미터다.
- 파일 작업은 임시 파일, 검증, 원자적 교체 순서를 따른다.
- 테스트 없이 핵심 도메인 로직을 완료 처리하지 않는다.
- 범위를 넓히는 기능은 먼저 제안하고 승인을 받는다.

## 검증 규칙

가능한 경우 아래를 모두 실행한다.

```text
cargo fmt --all -- --check
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
```

실패하면 원인, 영향 범위, 수정 결과를 보고한다. 테스트를 삭제하거나 경고를 숨겨 통과시키지 않는다.

## 작업 보고 형식

```md
## 변경 내용

## 변경 파일

## 검증 결과

## 승인 필요 여부

## 남은 작업
```
