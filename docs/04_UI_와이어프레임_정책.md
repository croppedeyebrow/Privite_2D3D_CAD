# UI 및 와이어프레임 정책

## 기본 화면

```text
┌─────────────────────────────────────────────────────────┐
│ 메뉴                 도구 모음                 프로젝트 │
├──────────────┬──────────────────────────────┬───────────┤
│ 도구/레이어  │                              │ 속성/검증 │
│              │          도면 캔버스          │           │
│              │                              │           │
├──────────────┴──────────────────────────────┴───────────┤
│ 명령 입력 / 프롬프트                         상태 표시 │
└─────────────────────────────────────────────────────────┘
```

## 영역 책임

- 상단 메뉴: 파일, 편집, 보기, 도구, 출력
- 도구 모음: 선택, 선, 사각형, 원, 치수, 텍스트, undo, redo
- 왼쪽 패널: 도구와 레이어
- 중앙 캔버스: 도면 표시와 포인터 입력
- 오른쪽 패널: 선택 객체 속성, 치수, 공차, 검증 결과
- 하단 명령 영역: 명령 입력, 좌표, 단위, snap 상태

## UI 상태 규칙

UI 상태와 도메인 상태를 분리한다.

- UI 상태: 선택 객체, 현재 도구, 포인터 위치, 확대율, 패널 상태
- 도메인 상태: Project, Drawing, Entity, Dimension, Tolerance

속성 패널에서 값을 수정할 때도 직접 필드 변경을 하지 않고 command를 발생시킨다.

## 상호작용 흐름

```text
도구 선택 -> 입력 수집 -> 미리보기 -> 확정 command -> 검증 -> 렌더 갱신 -> 자동 저장 예약
```

## UX 원칙

- 좌표와 단위를 항상 확인할 수 있어야 한다.
- 공차 계산 결과와 계산 trace를 숨기지 않는다.
- 오류는 대상 객체와 해결 방법을 함께 표시한다.
- 긴 작업 중에도 UI가 멈추지 않도록 작업 상태를 표시한다.
- 캔버스 조작과 속성 편집 모두 키보드로 접근 가능해야 한다.

## 금지 사항

- UI가 core model을 직접 변경하는 구현
- 화면에 보이는 값만 저장하는 구현
- 검증 없는 출력 버튼
- 오류를 무시하는 빈 catch 처리
- 사용자가 승인하지 않은 파일 삭제 또는 덮어쓰기
